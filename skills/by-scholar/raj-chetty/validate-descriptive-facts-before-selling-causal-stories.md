# Validate descriptive facts before selling causal stories

## Source Scholar

Raj Chetty

## Generalisable Skill

Treat large descriptive patterns as objects that need validation, robustness, and decomposition
before causal interpretation.

## Evidence Anchors

- Mobility maps
- Neighborhood effects papers
- Tax salience papers

## Taste Extraction Notes

### What the papers tend to test

The tests often compare across cohorts, siblings, movers, exposure duration, and alternative
definitions.

### What the abstract/introduction tends to do

The writing pattern separates 'what the data show' from 'what may explain it.'

### Why this skill travels

Apply this by adding a validation section before the mechanism section.

### Critical boundary

The limitation is that validated description may still not answer the policy counterfactual.

## Practice Exercise

Write the strongest descriptive fact in your project and two validation tests.

## How to Use This Skill in Your Own Research

1. Restate your research idea using this skill.
2. Name the theory or mechanism it implies.
3. State one empirical or theoretical test that could reject it.
4. Identify one rival explanation.
5. Write the introduction paragraph that would make the reader care.
