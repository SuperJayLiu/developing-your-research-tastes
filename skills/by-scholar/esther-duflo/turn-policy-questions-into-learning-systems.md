# Turn policy questions into learning systems

## Source Scholar

Esther Duflo

## Generalisable Skill

Design studies so each intervention teaches something about behavior, constraints, and
implementation, not only whether a program works.

## Evidence Anchors

- Using Randomization in Development Economics Research
- The Miracle of Microfinance?
- A Multifaceted Program Causes Lasting Progress for the Very Poor

## Taste Extraction Notes

### What the papers tend to test

The empirical tests often combine treatment effects with mechanisms, heterogeneity, take-up, and
implementation measures.

### What the abstract/introduction tends to do

The introduction typically motivates a real policy puzzle, then narrows it into a testable
intervention and learning objective.

### Why this skill travels

Use this skill when evaluating policy: specify the decision the study should inform and the
behavioral mechanism it should test.

### Critical boundary

The risk is program-specific learning that does not travel. Good taste requires explaining which
mechanism is portable and which implementation details are local.

## Practice Exercise

Write a program-evaluation question as both 'does it work?' and 'what would we learn if it works or
fails?'

## How to Use This Skill in Your Own Research

1. Restate your research idea using this skill.
2. Name the theory or mechanism it implies.
3. State one empirical or theoretical test that could reject it.
4. Identify one rival explanation.
5. Write the introduction paragraph that would make the reader care.
