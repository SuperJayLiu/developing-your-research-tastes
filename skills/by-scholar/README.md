# Skills by Scholar

Scholar-derived skills are now embedded directly inside each scholar page.

Why this structure:

- one scholar = one README;
- all copy/paste skills for that scholar are sections in the same page;
- no separate per-skill files under scholar folders;
- easier to browse on GitHub Pages and easier to maintain.

## Extraction Logic

Each embedded skill is extracted from:

1. what the scholar repeatedly treats as an important question;
2. what empirical/theoretical object they make central;
3. what they test, rule out, estimate, or formalize;
4. how their abstract/introduction frames the contribution;
5. what the skill can and cannot safely transfer to other projects.
