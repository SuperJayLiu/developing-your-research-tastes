# Skills

Skills are the central unit of the repo. A skill is a reusable research move, not a topic label, scholar tribute, or short note.

Examples include turning an anomaly into a research question, finding identification in institutions, building a minimal model of the core friction, separating risk from mispricing, measuring an unobservable concept, and designing a falsification test. Each mature skill should link to scholar pages, papers, journal tastes, and research steps.

The highest-quality skills should also behave like small agent-usable packages. The public chapter can stay readable, but the support layer should include progressive disclosure: a compact `SKILL.md`, reference files for detailed modules and gotchas, reusable templates under `assets/`, and scripts when a repeated check should be deterministic.

The first operational package is `nobel-research-taste/`. It turns the Nobel skill map into an agent-usable workflow with taste modules, gotchas, examples, evidence anchors, a card template, and a validator for mature skill-card structure.
