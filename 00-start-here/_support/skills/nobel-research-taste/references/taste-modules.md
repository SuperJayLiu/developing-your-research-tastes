# Nobel Taste Modules

This reference gives the operational modules behind the reader-facing Nobel skill cards. Use it to select the right taste lens for a live project. The longer book-style version is in `04-top-scholar-research-tastes/nobel-laureates/nobel-detailed-skill-cards.md`.

## Causal Design, Econometrics, And Measurement Discipline

Use this module when a project has data but the learning object is vague. The core move is to state the estimand, measurement object, or dynamic relation before choosing the method. A mature application names the treatment, comparison, outcome, population, counterfactual, and nearest threat. In time-series or econometric work, it names the process, shock, persistence object, identifying restriction, or measurement relation.

The key diagnostic is whether the method clarifies the question or replaces it. If the project says "we estimate the effect of X" but cannot say what counterfactual outcome is being compared, the skill has not been used. If the project presents robustness checks that do not attack the main threat, the skill is still immature.

## Formal Theory, Equilibrium, And Allocation

Use this module when a project has intuition but no disciplined economic object. The core move is to build the simplest benchmark, introduce one load-bearing departure, and derive a prediction, welfare comparison, or allocation result that would not exist without that departure.

The key diagnostic is whether removing the formal apparatus would change the paper's claim. If the model merely restates the introduction, it is decoration. If the benchmark already produces the result, the proposed friction is not doing work. A mature model makes the reader see which assumption carries the mechanism.

## Growth, Innovation, And Historical Change

Use this module when a project begins from a large long-run pattern: growth, development, technological diffusion, institutional persistence, creative destruction, climate constraints, or historical shocks. The core move is to convert the pattern into a mechanism carried by a state variable, incentive, technology, belief, capability, or organization.

The key diagnostic is whether the project separates persistence from explanation. A historical correlation can be a starting fact, but it is not yet a mechanism. The card should say why the effect survives and what comparison would make that channel credible.

## Institutions, Governance, And Political Economy

Use this module when a project has institutional detail but the detail is not yet doing analytical work. The core move is to translate the institution into a changed constraint: because rule X changes constraint Y for agent Z, behavior or outcome W should change.

The key diagnostic is whether the setting is scenery or mechanism. If the institutional paragraph contains many legal or political facts but no changed incentive, information condition, enforcement constraint, bargaining position, or participation rule, the project is not yet using institutional taste.

## Incentives, Information, Contracts, And Strategic Design

Use this module when agents know, hide, reveal, choose, promise, bargain, or strategize under constraints. The core move is to name the private object, write the binding constraint, and explain why the first-best or frictionless benchmark fails.

The key diagnostic is whether the strategic friction is load-bearing. If removing hidden information, hidden action, participation, limited liability, enforcement, or strategic interaction leaves the same prediction, the project is using incentive language rather than incentive design.

## Finance, Macroeconomics, And Market Signals

Use this module when a project interprets prices, returns, volatility, bank behavior, crises, monetary regimes, balance sheets, or macro-financial correlations. The core move is to define a benchmark and separate candidate interpretations before assigning a mechanism.

The key diagnostic is whether the evidence distinguishes risk, beliefs, liquidity, constraints, policy, cash-flow news, discount-rate variation, selection, and mispricing. A market pattern is not self-explanatory. A mature card makes interpretation slower and therefore stronger.

## Welfare, Behavior, Poverty, And Human Decision-Making

Use this module when a project studies household behavior, poverty, labor outcomes, development interventions, behavioral anomalies, consumption, welfare, or human constraints. The core move is to state the decision problem, constraint, intervention margin, and welfare object before making policy or human-behavior claims.

The key diagnostic is whether the project has a mechanism rather than a sympathetic outcome. Poverty, inequality, gender barriers, or behavioral mistakes can motivate a paper, but the research taste comes from showing which constraint changes and what welfare interpretation is justified.

## Markets, Trade, Allocation, And Design

Use this module when a project studies trade, matching, auctions, platforms, regulation, search, industrial organization, geography, or market design. The core move is to state the allocation object, the friction or design feature, and the welfare or distributional consequence.

The key diagnostic is whether the market is treated as an allocation system. If the project describes a market but cannot say what is allocated, who is matched with whom, what constraint binds, or how the rule changes welfare or distribution, the skill remains a topic label.
